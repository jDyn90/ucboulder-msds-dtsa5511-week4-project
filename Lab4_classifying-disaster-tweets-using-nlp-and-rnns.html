Project Background

This project involves building a model that analyzes users' social media posts on Twitter, and predicts which ones are referring to actual natural disasters happening in the area. This is a powerful tool that disaster relief organizations and news agencies would use to quickly understand what disasters are occurring in what areas, and gather information about them from people in the area observing these in real time. Many posts however may contain words that are are commonly used to describe disasters, such as "ablaze" or "fire", but not actually be referring to an emergency incident. The model we build will be able to reliably separate such posts from those referring to actual disasters with high accuracy.

10,000 tweets are collected for the purpose of model training and evaluation. In this Kaggle competition, the evaluation metric of choice is the F1 score which is computed as:

$$ \text{F1 Score} = 2 \times \frac{precision \times recall}{precision + recall} $$

Building the RNN Model

As the title indicates, we'll be using an RNN model for this classification task. An RNN model, as opposed to an ANN model, excels at processing sequential data including textual and temporal data. This is because RNNs have an internal memory that it builds while processing data, that it builds and refers to as it processes each sequential input. For our text data, this is great because the model will be able to consider the neighboring inputs and contextual clues, which are very important for understanding what each text is describing and its sentiment.

Embedding Layer
The first layer of our model will be an embedding layer. This layer will be responsible for taking textual inputs which we've converted to integer lists, and converting them into dense vectors in continuous space that are positioned in such a way that the distance between these vectors represents how similar or dissimilar they are to each other in meaning. This is vital for the model to be able to understand the meaning of texts, and consider them in the right contexts.

Recurrent Layer
The next layer will be the recurrent layer which is unique to any RNN model. We've learned of two important ones in lecture, the LSTM (Long Short Term Memory) and GRU (Gated Recurrent Unit) layers. We'll be building one RNN model with LSTM, and another with GRU, then comparing their performance to decide which architecture is better for this task.

Output Layer
The final layer will be a dense layer with a sigmoid activation function, so that it outputs a probability between 0 and 1 we will use for the actual classification task.

There's different hierarchies and hyperparameters we can tune to reduce overfitting. However, let's first start with simplifying the model. Earlier above we saw that our model has over 2.3 million trainable parameters, and this may just be too much for the data we have on hand. The embedding layer right now builds embeddings of dimension 100 - reducing this should lessen the number of parameters in the model, thus simplifying it. Using pre-trained embeddings (GloVe)

Instead of training the embedding layer from scratch, let's try using pre-trained embeddings instead. The goal of the embedding layer is to learn the semantic relationships between words to derive meaning. With only the ~8K samples in our dataset, the model may be learning a relationship that exists in the training dataset which isn't valid for any data outside of it. The benefit of pre-trained embeddings is that they're trained on a much larger corpus of texts, and so have a much better understanding of the true relationship between words that exists in the majority of text datasets.

One of the most popular pre-trained embeddings is GloVe (Global Vectors for Word Representation) developed by the Stanford NLP group. Embeddings are based on a co-occurrence matrix constructed for each word. See here for more info. GloVe makes available several datasets of embeddings of differing dimensions trained on various sizes of vocabulary. For this project we'll employ the one trained on 2b Twitter tweets, consisting of 1.2m vocabulary size and embeddings in 25d dimension.

In[40]:
# read in GloVe embeddings txt file, and save embeddings in a dict
embeddings_dict = {}

f = open('/kaggle/input/glove-6b-100/glove.6B.100d.txt', 'r')
for line in f:
    line_split = line.split()
    word = line_split[0]
    embedding = line_split[1:]
    embeddings_dict[word] = embedding
f.close()

RNN with LSTM Layer - Hyperparameter Optimization

Tuning hyperparameters will be done in a grid-search style on the following hyperparameters. Models will be built (following our LSTM architecture finalized above), and evaluated based on highest validation accuracy score reached during training.

Embedding Layer 'output_dim':

  • This controls the dimension of the embedding vector that our model learns for each word based on the data. A larger dimension requires more learnable parameters and implies a more complex model. Values we will try: [10, 25, 50, 75, 100]

LSTM Layer 'units':

  • This controls the dimension of the LSTM layer's output. A higher dimension requires more learnable parameters, and implies a more complex model learning more complex patterns. Values we will try: [8, 16, 32, 64, 128]

def build_eval_lstm_model(emb_output_dim, lstm_units):
    """
    This function trains a model with the same architecture as the base LSTM model we used, 
    but with custom values for a few hyperparameters. After training, it appends these parameters
    followed by the best-performing model's training and validation accuries. The best performing
    model is based on validation accuracy.

    Arguments:
        emb_output_dim (int) - value to use for parameter 'output_dim' in the Embedding layer
        lstm_units (int) - value to use for parameter 'units' in the LSTM layer

    Returns:
        None
    """
    model = Sequential()

    model.add(Input(
        shape = (texts_seq_pad.shape[1],) # length of the input vectors (i.e. encoded texts)
    ))
    
    model.add(Embedding(
        input_dim = len(tz.word_index) + 1 # size of the vocabulary.
        ,output_dim = emb_output_dim # length of the dense output vectors (i.e. embeddings).
    ))
    
    model.add(LSTM(
        units = lstm_units # dimensionality of the output space, should be set to a power of 2.
    ))
    
    model.add(Dense(
        units = 1
        ,activation = 'sigmoid' # to output a probability
    ))
    
    model.compile(loss = 'binary_crossentropy', optimizer = 'adam', metrics = ['accuracy'])

    es = EarlyStopping(verbose = 0, patience = 5, restore_best_weights = True)
    rl = ReduceLROnPlateau(verbose = 0, patience = 3, factor = 0.1)
    
    model_results = model.fit(x = texts_seq_pad, y = train['target'], epochs = 40, validation_split = 0.1, callbacks = [es, rl], verbose = 0)
    i = np.argmax(model_results.history['val_accuracy'])

    evals.append([emb_output_dim, lstm_units, model_results.history['accuracy'][i], model_results.history['val_accuracy'][i], model_results.history['loss'][i], model_results.history['val_loss'][i]])
    models.append(model) # save for later retrieval

Conclusions

We started with 7,613 samples of tweets containing words related to natural disasters. An RNN model using an LSTM layer was built & trained to classify tweets as being about actual natural disasters vs. not. The model is able to classify 88% of the training samples accurately, with a precision of 93% and recall of 87%.

Recall of 77% is lower than ideal

Recall is the portion of true samples in our data that the model correctly classified as true. In context, this means that the model did not recognize 17% of tweets that were actually referring to real disasters. Precision was high at 95%, and so this may imply the model is too conservative in its predictions. Emergency responders and news agencies definitely need a model that can identify near 100% of disaster-related tweets, and missing 17% of them will mean that some disasters go unnoticed for longer than necessary. On the other hand, the AUC for our model is 0.96 which is very high, so there may not be much room to improve. We also don't want to increase false positive rate too much when trying to increase recall, as that will lead to consumers of the model wasting time investigating tweets that aren't actually about disasters.

Attempts to improve model performance with pre-trained embeddings was not successful

Using the GloVe pre-trained embeddings actually led to lower performance in terms of accuracy and loss metrics, even though there were less signs of overfitting than when using learned embeddings. This was surprising to me since pre-trained embeddings were trained on a much larger and comprehensive dataset than what's in our training data, and so should capture the true semantic relationship between words much better and help the model. It seems to be that using custom learned embeddings was better for this particular problem. It would be useful to try other pre-trained embeddings such as Word2Vec, and explore using pre-trained models such as BERT.

Attempts to improve model performance with architecture changes was not successful

I tried using dropout layers, and switching LSTM to GRU layer but none were able to outperform the base model that I ended up using for the final predictions. This was also surprising because in other notebooks I reviewed, adding dropout did improve model performance on the validation set. There's definitely more architectural changes I can try such as using Spatial Dropout instead of "vanilla" dropout layers, and using dropout within the LSTM layer itself which is an available functionality in Keras.
